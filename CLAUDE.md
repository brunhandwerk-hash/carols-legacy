# Carol's Legacy

Browser RTS about Sinaia, Romania (1690‚Äì1947): the town grows from forest hamlet to royal summer residence. Three.js + TypeScript + Vite, no game engine, all geometry procedural and all textures generated at runtime (no binary asset files).

## Commands
- `npm run dev` ‚Äî Vite dev server (port 5173)
- `npm run build` ‚Äî type-check (`tsc --noEmit`) + production build

## Workflow
- **Commit successful implementations**: once a change is complete and verified (`npm run build` passes, behaviour confirmed), commit it to `main` with a descriptive message. Don't leave finished, working features sitting uncommitted in the working tree.

## Design rules
- **History first**: landmark buildings stand on fixed plots ([src/plots.ts](src/plots.ts)) matching real Sinaia geography. Dates/events come from [docs/research-sinaia.md](docs/research-sinaia.md) ‚Äî check it before adding eras or buildings.
- **Conflict** is historical events only (WWI 1916 defense chapter), never a rival AI base.
- **Art style**: grounded **stylized realism** (reference: Manor Lords / Foundation / Valheim) ‚Äî believable building proportions and materials, not flat boxes. Use **`MeshStandardMaterial` (PBR)** lit by an environment map, **ACES Filmic tone-mapping** + sRGB output, soft shadows, and atmospheric fog. Surfaces get tactile **procedurally-generated textures + normal maps** (canvas noise at runtime ‚Äî wood, stone, thatch, plaster). Terrain keeps vertex colors but gains a subtle procedural normal/roughness. Avoid plain untextured `MeshLambertMaterial`. Shared building materials live in `M` ([src/buildings.ts](src/buildings.ts)); shared texture/material helpers in `src/materials.ts`. Structural masses use the chamfered `box()` helper (RoundedBoxGeometry) so edges catch light instead of reading as hard cubes.
  - **Post-processing** ([src/main.ts](src/main.ts)): an `EffectComposer` chain ‚Äî `RenderPass` ‚Üí `GTAOPass` (ground-contact ambient occlusion, world-space radius so distant peaks are unaffected) ‚Üí subtle `UnrealBloomPass` ‚Üí `OutputPass` (does the ACES + sRGB). Renders into a HalfFloat 4√óMSAA target. The frame is drawn via `composer.render()`, not `renderer.render()`.
  - **Asset policy (hybrid, relaxed further)**: image texture files AND authored glTF models are now permitted. **All buildings currently render CC0 KayKit glTF models** ([public/models/kaykit/](public/models/kaykit/), wired via `BuildingDef.model` + `Building.loadHeroModel`); the procedural builders in `buildings.ts` remain as silent fallbacks if a `.glb` is missing. Image textures live under `public/textures/` (ground splat + foliage atlas) and `public/hdri/` (sky). Several KayKit‚Üíbuilding mappings are placeholders that don't 100% fit (e.g. monastery‚Üícastle, st√¢nƒÉ‚Üíwatchtower) ‚Äî to be replaced with better/hero models later. Era-reskinning of glTF buildings is not yet implemented (future follow-up).
- UI language: English.

## Architecture
- `src/terrain.ts` ‚Äî terrain from the **real Sinaia DEM** (`public/dem.bin` + `dem.json`, baked by `node scripts/fetch-dem.mjs` from AWS terrarium tiles, bbox 45.32‚Äì45.385N / 25.50‚Äì25.58E, 1 unit = 1 m, heights relative to 745 m a.s.l.). `loadDem()` must complete before anything samples terrain (async `boot()` in main.ts). `terrainHeight(x,z)` is the single source of truth for gameplay; plots are flattened via `registerFlatSpot` *before* the mesh is built; the Prahova course is auto-traced along the DEM's valley floor. Landmarks are georeferenced lat/lon in `src/plots.ts` (`initPlots()` converts to world coords). **`surfaceHeight(x,z)`** reconstructs the *rendered* coarse mesh's triangle (the mesh is far lower-res than the DEM) ‚Äî villagers and buildings sit on it so they don't sink/float; gameplay (slope, pathing) still uses `terrainHeight`. `buildRoadMesh()` drapes a cobbled ribbon along the `ROADS` polylines. A wide-area low-res **backdrop DEM** (`public/backdrop.bin` + `backdrop.json`, baked by `node scripts/fetch-backdrop.mjs`, bbox ~45.20‚Äì45.52N / 25.34‚Äì25.74E) is rendered by `buildBackdropMesh()` as the real surrounding massifs (Bucegi W/SW up to Omu, BƒÉiului E) ringing the playable map with the same lon/lat‚Üíworld projection (central footprint left hollow for the detailed mesh). It needs the far camera plane (70 km, main.ts) + long fog (world.ts).
- `src/materials.ts` ‚Äî runtime-generated PBR materials (albedo+normal maps drawn from value noise; wood/stone/brick/thatch/tile/plaster/cobble). No image files.
- `src/world.ts` ‚Äî scene, lights, backdrop peaks, instanced trees (no shadows)/rocks/bushes; each instance is a `ResourceNode` in `G.nodes`. PBR pipeline (ACES tone-map, env map) is set up in `main.ts`.
- `src/state.ts` ‚Äî global mutable state `G` (resources incl. `coin`, villagers, buildings, era). Exposed as `window.G`; `window.scene`/`rig`/`terrain` also exposed for debugging.
- `src/buildings.ts` ‚Äî `BuildingDef` registry + procedural mesh builders (gable roofs, footings, chimneys, framed openings); phases planned ‚Üí site ‚Üí done. Each building terraces its ground (`addFoundation`), can be `demolish()`ed, and **re-skins by era** (`eraStyle` bands: log/thatch ‚Üí plaster/tile ‚Üí brick/slate; `reskinAllBuildings()` on era advance; builders `tag()` wall/roof meshes). **Production chains**: a def's `produces` ({input, output, interval}) consumes raw resources into refined goods on a timer (Sawmill wood‚Üíplanks, Stonecutter stone‚Üíblock); refined goods (`planks`, `block` in `ResKind`) gate landmark costs. **Worker assignment**: a def with `jobSlots` only produces while assigned villagers are present, and output scales with how many (`Building.workers`/`assignWorker`/`presentWorkers`; villager `work` task; right-click a workplace to assign). Buildings without `jobSlots` (coin trickles) stay passive. Optional `model` swaps a procedural landmark for an authored glTF ([src/models.ts](src/models.ts)).
- `src/units.ts` ‚Äî villager task state machine (idle/move/gather/build/work/shelter/fight); each villager has a `Profession` with a distinct outfit + tool. Movement uses A* (`navToward` ‚Üí [src/pathfind.ts](src/pathfind.ts)) to route around buildings/slopes. Gather-camp workers auto-harvest nearby nodes (`campGather`).
- `src/population.ts` ‚Äî **Banished-style population**: villagers eat from the food stockpile; with a surplus + free housing (`pop < popCap`) the town grows on its own; an empty larder starves villagers. No manual "train settler". Houses raise `popCap`.
- `src/labor.ts` ‚Äî **Banished-style auto-assignment**: idle villagers self-assign to the nearest building with an open job (`Building.openPositions()` = `desired` ‚àí assigned) or a construction site short of builders. You set `desired` per building (+/‚àí in the panel); you don't command individuals. Workplaces (gather camps, sawmill/stonecutter, food buildings) only produce while staffed; output scales with present workers. Called on a timer from the main loop.
- `src/eras.ts` ‚Äî era objectives & progression; spawns 'planned' signposts per era; calls `reskinAllBuildings()` on advance.
- `src/input.ts` ‚Äî RTS camera, click/box selection, right-click commands, ghost placement.
- Coordinates: +x east, +z south, metres; map 6258√ó7224 m centred on the bbox. Don't hardcode world positions ‚Äî derive them from lat/lon via `lonLatToWorld`.
